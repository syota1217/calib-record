# ループ校正記録 PWA - 導入・使用ガイド

スマホのホーム画面にアイコンを置いてアプリ風に使える「ループ校正記録」のセットアップ手順です。

---

## 📦 同梱ファイル

```
calib_pwa/
├─ index.html                 アプリ本体 (メインページ)
├─ manifest.json              PWAマニフェスト (アプリ名・アイコン定義)
├─ sw.js                      Service Worker (オフライン動作用)
├─ icon-192.png               ホーム画面アイコン (192px)
├─ icon-512.png               ホーム画面アイコン (512px)
├─ icon-maskable-512.png      Android用マスカブルアイコン
└─ README.md                  このファイル
```

---

## ⚙️ セットアップ (配信者側の作業)

PWAとして動作させるには **HTTPSで配信されたWebサーバー** が必要です (PWA仕様)。以下のいずれかの方法で配置してください。

### 方法A: 社内サーバー (Apache / Nginx / IIS)

1. 社内Webサーバーの公開ディレクトリに `calib_pwa` フォルダをアップロード
2. HTTPSで配信されていることを確認 (例: `https://intra.company.local/calib_pwa/`)
3. ブラウザでアクセスできればOK

### 方法B: GitHub Pages (無料・おすすめ)

1. GitHubアカウントでリポジトリを新規作成 (例: `calib-record`)
2. 同梱のファイルをすべてコミット・プッシュ
3. リポジトリの `Settings` → `Pages` → Source: `main` branch を選択
4. 数分後、`https://<ユーザー名>.github.io/calib-record/` でアクセス可能に
5. HTTPS配信・無料・自動更新なので一番ラク

### 方法C: ローカルPCで動作確認のみ

Pythonが入っていれば、ローカルで簡易サーバーを起動して確認できます。
(※ ただしスマホからは同じWi-Fi内でしかアクセスできず、iOSではHTTPS不要の `localhost` でも Service Worker が動作しないことがあります)

```bash
cd calib_pwa
python3 -m http.server 8080
```

ブラウザで `http://localhost:8080` にアクセス。

## 📱 スマホへのインストール方法

### iPhone (iOS 16.4 以降 Safari 必須)

1. **Safari** で配信URLにアクセス (Chromeでは不可)
2. 画面下の **共有ボタン** (□から↑が出ているアイコン) をタップ
3. メニューを下にスクロールして **「ホーム画面に追加」** をタップ
4. アプリ名 (校正記録) を確認して「追加」をタップ
5. ホーム画面にアイコンが追加される
6. アイコンをタップするとアドレスバーなしのフルスクリーンで起動

### Android (Chrome推奨)

1. **Chrome** で配信URLにアクセス
2. 画面下に「アプリをインストール」のバナーが自動表示される
   - → 「インストール」をタップ
3. バナーが出ない場合:
   - 右上のメニュー (⋮) → **「アプリをインストール」** または **「ホーム画面に追加」**
4. ホーム画面にアイコンが追加される

### PCでも使える (Chrome / Edge)

1. ChromeまたはEdgeで配信URLにアクセス
2. アドレスバー右端の **⊕** アイコン (インストール) をクリック
3. デスクトップやスタートメニューにアプリとして追加される

## ✨ PWAの特徴

- **オフライン動作**: 初回アクセス後は通信なしでも動作
- **自動保存**: 入力内容はブラウザ内に保存され、再起動しても復元される
- **ブラウザUIなし**: 起動時はアドレスバー等が消え、純正アプリのように全画面表示
- **自動更新**: サーバー側のファイルを更新すれば、次回起動時に最新版が反映される
- **データ送信なし**: 計算はすべて端末内で実行、外部への送信は一切なし

---

## 🔧 データ更新時 (配信者向け)

アプリのコードを更新してユーザーに反映させたい場合:

1. `index.html` を編集
2. `sw.js` のバージョン番号を変更 (例: `v1.0.0` → `v1.0.1`)
   ```js
   const CACHE_NAME = 'calib-record-v1.0.1';  // ← ここを変える
   ```
3. サーバーに再アップロード
4. ユーザーが次回起動時に自動で新しいキャッシュに差し替わる

## 🛠 トラブルシューティング

| 現象 | 原因と対処 |
|---|---|
| ホーム画面に追加が出てこない | HTTPSで配信されていない / iOSの場合 Safari以外を使っている |
| オフラインで起動しない | 一度もオンラインでアクセスしていない → 先に1回オンライン接続で起動 |
| 更新が反映されない | `sw.js` のバージョン番号を変更し忘れている / ブラウザのキャッシュをクリア |
| アイコンが丸く切られる (Android) | 仕様通り。`icon-maskable-512.png` がsafe zoneに収まるよう設計済み |
